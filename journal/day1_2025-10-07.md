- **Cause:** Unicode emojis (‚úÖ ‚ùå) can't encode to charmap

- **Fix:** Replaced 2,684+ emojis across 76 Python files with ASCII ([OK], [X])

#### Bug #2: API Signature Error - HIGH SEVERITY ‚úì FIXED
- **Impact:** `check_account_status.py` crashed every time
- **Cause:** Wrong API signature for `get_orders()`
- **Fix:** Used `GetOrdersRequest` with proper enums
- **Status:** RESOLVED

#### Bug #3: Failed Trades Count Against Limit - CRITICAL ‚úì FIXED
- **Impact:** Failed trades wasted daily trade slots
- **Cause:** `trades_today.append()` outside try block
- **Fix:** Moved inside try, added `continue` in except
- **Status:** RESOLVED - Tomorrow all 5 slots available

#### Bug #4: Timezone Handling - CRITICAL ‚úì FIXED
- **Impact:** Scanner would run at wrong times if not in PDT timezone
- **Cause:** Using `datetime.now()` without timezone awareness
- **Fix:** Added `pytz`, forced America/Los_Angeles timezone
- **Status:** RESOLVED - Works from any timezone now

#### Bug #5: Time Import Inside Loop - MEDIUM ‚úì FIXED
- **Impact:** Inefficient code, import overhead on every trade
- **Cause:** `import time` inside try block (line 227)

## Lessons Learned

### Technical Lessons
1. **Always test on Windows console first** - Emoji encoding is a real issue on Windows
2. **API documentation matters** - Wrong signatures crash programs
3. **Order verification is essential** - AES bug would've gone unnoticed without logging
4. **Timezone independence is critical** - System needs to work from anywhere
5. **Failed trades shouldn't count** - Wasting slots kills performance

### Trading Lessons
1. **Position sizing matters** - 45% allocations were too aggressive for $100k account
2. **Buying power management** - Reserve capacity for all trade legs
3. **Paper trading has bugs too** - Alpaca's 21x fill proves even simulators fail
4. **Fallback strategies work** - AAPL stock purchase saved the trade
5. **High-confidence opportunities exist** - 503 stocks scored 4.0+ consistently

### Strategic Lessons
1. **Build incrementally** - Week 2 Day 1 is about proving execution, not max returns
2. **Fix bugs immediately** - All 4 bugs fixed same day prevents compounding issues
